Architecture

How SupportAI Works

1

Ticket Intake & Context Gathering

The environment passes an incoming customer request to the agent, providing full historical context, user sentiment, and relevant account details.

2

Classification Action

The agent uses the classify action to assign the ticket to one of the 10 distinct workflow types. It receives a reward based on classification accuracy.

3

Resolution & Empathy

The agent outputs a response. Our deterministic grader analyzes the text for specific resolution signals and empathy markers to assign a quality score.

4

Escalation or Closure

Depending on the workflow rules, the agent must either correctly escalate complex issues, or close the ticket upon successful resolution, completing the episode.

Knowledge Base

Frequently Asked Questions

How is the reward calculated?

The environment gives intermediate rewards for correct classification (+0.7), and terminal rewards based on text quality, empathy, and correct termination logic (escalate vs close).

Can I use my own models with this environment?

Yes! The environment is strictly API-driven following standard Gym/PettingZoo paradigms. Any local language model or remote API can interact via the /step endpoint.

Is there a multi-agent mode?

We are currently developing a v2 release which supports primary agents delegating to specialized sub-agents alongside a supervisor model.
